Call Handling Workflows

A. Outgoing Calls (CRM-Initiated)

Complete Outgoing Call Flow - From Agent Click to CRM Update

Step 1: Agent clicks "Call" button in CRM for selected customer
Step 2: CRM sends API request to PBX server with customer details
Step 3: PBX checks available SIM pool and selects optimal SIM
Step 4: Call routed through GSM Gateway using selected SIM
Step 5: Customer's mobile rings and call connects
Step 6: Call automatically recorded and logged with metadata
Step 7: CRM updates call status and recording link automatically
Key Features:
  • Manual or automatic SIM selection based on usage patterns
  • Recording starts automatically with full metadata capture
  • Real-time call status updates to CRM
  • Intelligent SIM rotation to prevent blocking

B. Incoming Call Management

Incoming Call Flow - Professional Call Handling with CRM Integration

Step 1: Customer calls any of your enabled SIM numbers
Step 2: GSM Gateway receives call and forwards to PBX
Step 3: PBX identifies the dialed number (DID)
Step 4: Call routed to appropriate queue or agent based on rules
Step 5: CRM auto-popup displays customer record (if existing)
Step 6: Call recording enabled and properly tagged

Optional: High Availability (HA) Setup

For mission-critical operations, we recommend an Active-Passive High Availability configuration:

  • Primary PBX server handles all active calls
  • Secondary server in standby mode with real-time synchronization
  • Automatic failover in case of primary server failure
  • Minimal downtime (typically under 30 seconds)
  • Shared storage for call recordings and configuration
